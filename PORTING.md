# notebooklm-py -> notebooklm-api (Rust) Porting Map

## Scope for phase 1 (implemented)
- Project scaffolding for both library crate and CLI binary.
- Authentication from Playwright `storage_state.json`.
- Token fetch (`SNlM0e`, `FdrFJe`) from NotebookLM homepage.
- Batchexecute RPC request/response encode/decode pipeline.
- Notebook operations:
  - `list`
  - `create`
  - `get`
  - `rename`
  - `delete`
  - `get_summary`
- Sources operations:
  - `list`
  - `get`
  - `add_url`
  - `rename`
  - `delete`
  - `refresh`
  - `get_fulltext`
- Chat operations:
  - `ask`
  - `get_history`
  - local cached conversation turns for follow-up context
- Research operations:
  - `start` (web/drive, fast/deep validation)
  - `poll`
  - `import_sources`
- Settings operations:
  - `get_output_language`
  - `set_output_language`
- Sharing operations:
  - `get_status`
  - `set_public`
  - `set_view_level`
  - `add_user`
  - `update_user`
  - `remove_user`
- Artifacts operations:
  - `list`
  - `get`
  - `generate_audio`
  - `generate_video`
  - `generate_report`
  - `generate_quiz`
  - `generate_flashcards`
  - `generate_infographic`
  - `generate_slide_deck`
  - `generate_data_table`
  - `generate_mind_map`
  - `download_audio`
  - `download_video`
  - `download_infographic`
  - `download_slide_deck`
  - `download_report`
  - `download_data_table`
  - `download_quiz` (json/markdown/html)
  - `download_flashcards` (json/markdown/html)
  - `export`
  - `export_report`
  - `export_data_table`
  - `poll_status`
  - `wait_for_completion`
  - `rename`
  - `delete`
- CLI commands:
  - `notebooklm auth-status`
  - `notebooklm notebook ...`
  - `notebooklm source ...`
  - `notebooklm artifact ...`
  - `notebooklm research ...`
  - `notebooklm settings ...`
  - `notebooklm share ...`
  - `notebooklm chat ...`
  - `notebooklm chat-history ...`
  - backward-compatible aliases: `notebooklm list`, `notebooklm create <title>`
- Parser/unit test coverage:
  - RPC decode sanity test
  - share status parsing
  - interactive app-data extraction
  - data-table structure parsing
  - CSV escaping
  - generation result parsing
  - completed-artifact selection
  - HTTP error taxonomy mapping (auth/rate-limit/server)
- Reliability hardening:
  - Automatic one-shot auth refresh + retry on 401/403 for RPC/query/download paths
  - HTTP status mapping to typed errors (auth/rate-limit/client/server)
- Typed API ergonomics:
  - Sharing enums for access/view-level/permission
  - Research enums for source/mode
  - Artifact enums for interactive output format and export type
  - Typed artifact generation option models and CLI value-enum flags
  - Mind-map output format enum and note-backed generation persistence
- Parsing and fixtures:
  - Centralized JSON path helpers with decode-shape errors
  - Fixture-backed parser tests (notebooks/share/data-table)
- Retry and error hardening:
  - Configurable retry policy with backoff+jitter for RPC/query/download
  - Expanded errors: timeout/network/stale-auth/decode-shape
- CLI UX:
  - Global `--output` (`json|table|tsv`) and `--quiet`

## Python -> Rust module mapping
- `notebooklm/auth.py` -> `src/auth.rs`
- `notebooklm/rpc/{types,encoder,decoder}.py` -> `src/rpc.rs`
- `notebooklm/client.py` + `notebooklm/_notebooks.py` -> `src/client.rs`
- `notebooklm/types.py::Notebook` -> `src/types.rs`
- `notebooklm/notebooklm_cli.py` -> `src/bin/notebooklm.rs`

## Next features to port
- Additional fixture/cassette parity against notebooklm-py tests.
